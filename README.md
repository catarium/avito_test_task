## Принятые решения не определенные в спецификации
* при обновлении пользователей (указано в спецификации) в случае если у пользователя меняется команда и у него есть PR или он ревьюит PR, они у него остаются
* PR всегда возвращает createdAt и mergedAt, т.к. они были указаны в спецификации
* getReview возвращает NOT_FOUND если пользователь не найден
